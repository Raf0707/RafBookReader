# ТЗ: перевод всей книги в читалке

## Краткое описание

Нужно добавить в читалку функцию перевода всей книги. Пользователь открывает книгу, нажимает кнопку перевода, выбирает исходный и целевой язык, после чего приложение переводит текст по абзацам и показывает результат прямо в Reader.

Перевод выполняется через Google ML Kit on-device translation. Результат сохраняется в кэш на устройстве, чтобы при повторном открытии той же книги и выборе той же языковой пары не переводить всё заново.

## Основной сценарий

1. Пользователь открывает книгу в Reader.
2. В верхней панели Reader появляется кнопка перевода.
3. После нажатия открывается диалог или bottom sheet выбора языков:
   - исходный язык;
   - целевой язык.
4. Пользователь запускает перевод.
5. Приложение показывает состояние загрузки:
   - при необходимости сначала сообщение `Загрузка языковой модели`;
   - сообщение `Выполняется перевод книги`;
   - прогресс в процентах;
   - возможность отменить перевод.
6. Книга переводится по абзацам.
7. После завершения Reader начинает показывать переведённый текст, не изменяя исходный список элементов книги.
8. Пользователь может вернуть отображение оригинального текста без повторной загрузки книги.

## Провайдер перевода

Для первой версии используется Google ML Kit Translation:

- работает внутри Android-приложения;
- не требует API-ключа;
- не требует отдельного backend;
- поддерживает загрузку языковых моделей на устройство;
- подходит для дипломной реализации без платных внешних API.

В `app/build.gradle.kts` нужно добавить зависимость:

```kotlin
implementation("com.google.mlkit:translate:17.0.3")
```

При реализации можно обновить версию до актуальной стабильной версии ML Kit Translation, если она совместима с проектом.

Внешние сервисы вроде DeepL или Google Cloud Translate в первой версии не используются, потому что для них нужен API-ключ. Хранить такой ключ прямо в APK небезопасно, поэтому для них потребовался бы отдельный сервер/proxy.

## Поведение перевода

- Переводить только текстовые элементы книги:
  - обычные абзацы;
  - названия глав.
- Не переводить и сохранять как есть:
  - изображения;
  - разделители;
  - математические формулы.
- Перевод выполнять последовательно по элементам текста, чтобы не отправлять всю книгу одним большим запросом.
- После каждого переведённого элемента обновлять прогресс.
- Перед переводом вызвать `downloadModelIfNeeded` для нужной языковой пары.
- Пока скачивается модель, показывать отдельный статус `Загрузка языковой модели`.
- Если модель не скачалась из-за отсутствия сети, нехватки места или другой ошибки, показать понятное сообщение и оставить оригинальный текст.
- После успешной загрузки модели перейти к статусу `Выполняется перевод книги`.
- Если пользователь отменяет перевод, оригинальный текст остаётся без изменений.
- Если возникает ошибка, показать сообщение и оставить оригинальный текст.

### Сохранение структуры и стилей

Исходный список `ReaderText` нельзя менять по структуре:

- количество элементов должно остаться тем же;
- порядок элементов должен остаться тем же;
- `Image`, `Separator` и `Math` должны переноситься в результат без изменений;
- у `ReaderText.Chapter` нужно переводить только `title`, сохраняя `id` и `nested`;
- у `ReaderText.Text` нужно переводить `line.text`.

Так как `ReaderText.Text` хранит `AnnotatedString`, реализация должна стараться сохранять стили:

- при сборке переведённого `AnnotatedString` переносить существующие paragraph styles и span styles, если это можно сделать безопасно;
- если после машинного перевода длина или структура текста не позволяет корректно перенести inline ranges, использовать plain translated text;
- невозможность перенести часть стилей не должна приводить к падению приложения или отмене всего перевода.

Стопроцентное сохранение inline-стилей после машинного перевода в первой версии не гарантируется, но безопасные случаи должны сохраняться.

### Отмена перевода

Перевод должен выполняться в отдельном coroutine `Job`.

При отмене:

- отменить активный `Job`;
- остановить обработку между элементами текста;
- не применять частичный результат к Reader;
- не сохранять частичный результат как готовый кэш;
- оставить на экране оригинальный текст;
- закрыть `Translator`, если он уже был создан.

## Кэширование

Кэш перевода хранить в файловой системе приложения, например:

```text
application.filesDir/book_translations/
```

Ключ кэша должен учитывать:

- `bookId`;
- нормализованный путь к файлу книги или hash пути;
- исходный язык;
- целевой язык.

Дополнительно кэш должен учитывать fingerprint файла книги:

- минимум `fileSize + lastModified`;
- предпочтительно hash содержимого файла, если его можно получить безопасно и без заметной задержки.

Если fingerprint текущего файла не совпадает с fingerprint в кэше, такой кэш использовать нельзя.

Формат кэша: JSON-файл с переведёнными текстовыми элементами по индексам исходного списка `ReaderText`, языковой парой и fingerprint исходного файла.

Room-таблицы для первой версии менять не нужно. Оригинальная книга и `BookEntity` не должны изменяться.

## Изменения в архитектуре

Добавить слой перевода:

- `BookTranslationRepository` для чтения и записи кэша;
- реализацию репозитория в data layer;
- use case `TranslateBook`;
- Hilt binding для нового репозитория.

`TranslateBook` должен:

- принимать исходный `List<ReaderText>`;
- принимать языковую пару;
- принимать идентификатор книги и fingerprint файла для проверки кэша;
- проверять/использовать кэш;
- при отсутствии кэша запускать ML Kit translator;
- скачивать модель через `downloadModelIfNeeded`;
- переводить элементы по очереди;
- сохранять успешный результат в кэш;
- не сохранять результат при отмене или ошибке;
- закрывать `Translator` после завершения.

## Изменения в Reader

В `ReaderState` добавить:

- исходный текст книги `originalText: List<ReaderText>`;
- переведённый текст книги `translatedText: List<ReaderText>?`;
- выбранный исходный язык;
- выбранный целевой язык;
- флаг выполнения перевода;
- прогресс перевода;
- статус перевода, например загрузка модели или перевод текста;
- ошибку перевода;
- флаг отображения переведённого текста.

Отображаемый в Reader список должен вычисляться по флагу отображения перевода:

- если `showTranslatedText == true` и `translatedText != null`, показывать `translatedText`;
- иначе показывать `originalText`.

Возврат к оригиналу должен переключать флаг отображения, а не заново читать книгу с диска.

В `ReaderEvent` добавить:

- `OnShowBookTranslationDialog`;
- `OnStartBookTranslation`;
- `OnCancelBookTranslation`;
- `OnRestoreOriginalText`.

В `ReaderModel` добавить обработку:

- запуска перевода;
- отмены перевода;
- применения переведённого текста;
- восстановления оригинального текста;
- чтения и записи кэша;
- сохранения текущей позиции чтения при переключении оригинал/перевод.

В UI добавить:

- кнопку перевода в `ReaderTopBar`;
- диалог/bottom sheet выбора языков;
- overlay или modal progress во время перевода;
- действие для возврата к оригинальному тексту.

## Языки

Для первой версии использовать только языки, которые уже есть в приложении и поддерживаются ML Kit.

Список языков для выбора нужно строить программно:

- взять список языков приложения;
- проверить поддержку через `TranslateLanguage.fromLanguageTag(...)` или `TranslateLanguage.getAllLanguages()`;
- скрыть языки, которые ML Kit Translation не поддерживает.

Языки приложения `be`, `tt`, `ba` должны быть скрыты, если ML Kit их не поддерживает.

Ожидаемый список для первой версии:

- English;
- Українська;
- Deutsch;
- العربية;
- Español;
- Türkçe;
- Français;
- Polski;
- Русский;
- Italiano;
- 中文;
- हिन्दी;
- Português.

Автоопределение языка в первой версии не добавлять, чтобы не подключать дополнительный ML Kit Language Identification API.

## Проверка

Нужно проверить следующие сценарии:

- книга открывается как раньше без запуска перевода;
- пользователь запускает перевод, видит загрузку модели, затем прогресс перевода и получает переведённый текст;
- после успешного перевода текущая позиция чтения не сбрасывается;
- повторный перевод той же книги с той же языковой парой берётся из кэша;
- изменённый файл книги не использует старый кэш из-за несовпадения fingerprint;
- отмена перевода оставляет оригинальный текст;
- отмена перевода не записывает частичный результат как готовый кэш;
- при ошибке загрузки модели или перевода показывается ошибка;
- изображения, разделители и формулы остаются на месте;
- `Chapter.id`, `Chapter.nested`, порядок элементов и количество элементов Reader сохраняются;
- стили `AnnotatedString` сохраняются там, где перенос безопасен;
- если перенос стилей невозможен, приложение показывает plain translated text и не падает;
- прогресс чтения и настройки Reader не сбрасываются.

## Ограничения первой версии

- Перевод не создаёт новую книгу в библиотеке.
- Оригинальный файл книги не изменяется.
- Перевод хранится только в кэше приложения.
- Перевод выполняется только при явном запуске пользователем.
- DeepL и Google Cloud Translate не используются.
